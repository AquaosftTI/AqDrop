Saudações!

É com prazer que disponibilizamos o Aquasoft DROP, um framework desenvolvido para encorajar o desenvolvedor a estudar e utilizar todo o potencial da Orientação a Objetos do Delphi.

O DROP é totalmente aberto. Isto quer dizer que, mantendo o direito autoral da Aquasoft, você poderá utilizar o DROP para qualquer tipo de projeto.

Descrição dos pacotes que compõem o DROP:
  - Core: contém bibliotecas genéricas do Drop, como tipos básicos, collections, pattenrs, entre outras classes;
  - DB: contém as classes que servem como base para comunicação com SGBDs, entre elas, as classes de conexão, e de ORM;
  - DBX: contém as especializações necessárias para comunicação com SGBDs via DBX framework;
  - FD: contém as especializações necessárias para comunicação com SGBDs via FireDAC;
  - Register: pacote de Design Time para registrar a marca Drop no IDE.

O DROP não precisa ser instalado no IDE, basta adicionar os fontes do DROP ao seu projeto. Mas se você quiser registrar o DROP no Delphi, basta seguir estes passos:
  - Adicione as pastas que contêm os fontes de runtime à sua Library Path (pasta Core e pasta DB); 
  - Abra o grupo de projetos específico para a sua versão de Delphi (por exemplo, AqDrop.DXE5.groupproj para o Delphi XE5);
  - Execute o build em todos os pacotes;
  - Instale o pacote Register.

NOTAS IMPORTANTES:
  - Para agilizar a sua liberação, a versão 1.4 do DROP está sendo publicada sem a atualização dos pacotes de RT e DT. Esperamos atualizar os pacotes em breve, bem como criar os pacotes para o Delphi Rio (10.3). Se você não usa pacotes para modularizar a sua aplicação, você não será afetado. No entanto, se você precisa dos pacotes atualizados para sua aplicação, entre em contato pelo e-mail tatu@taturs.com que agilizaremos a entrega dos pacotes necessários;
  - O Drop é disponibilizado na modalidade 'As Is', isto significa que a Aquasoft não se responsabiliza por eventuais problemas decorrentes da utilização do Drop, bem como não é obrigada a implementar / modificar a ferramenta para se adaptar ao cenário de qualquer usuário do Drop;
  - Na teoria, os fontes do projeto são compatíveis com o Delphi XE3, mas a partir da versão 1.2, estamos dando suporte oficial (via testes unitários) somente para versão XE4 e mais recentes;
  - O Drop contempla classes especializadas para conexões com Interbase, MSSQL, MySQL, Firebird, Oracle, SQLite e Postgres (este último somente via FD). Fique à vontade para sugerir outros SGBDs;
  - Pacotes para o Delphi XE8 foram incluídos na versão 1.2 do DROP, no entanto, uma regressão gerou um problema na compilação dos pacotes, regressão esta que foi corrigida nas versões seguintes da ferramenta. Se você não modulariza suas aplicações via BPLs, este problema não lhe causará nenhum efeito, pois a inclusão dos fontes (via projeto ou library path) em executáveis trará o comportamento correto do DROP (garantido via bateria de testes na versão XE8);
  - Mantenha-se atualizado sobre as novidades do DROP através do nosso twitter e facebook: TatuRS ou AquasoftTI;
